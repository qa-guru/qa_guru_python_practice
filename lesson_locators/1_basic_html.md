# Часть 1. Основы: DOM, элементы и локаторы

## HTML‑элементы
Любая веб‑страница состоит из HTML‑элементов. Каждый элемент — это часть интерфейса: кнопка, поле ввода, текстовый блок, изображение, модальное окно и другие компоненты.

HTML‑элементы описываются с помощью тегов:

```html
<p>Hello world!</p>
<button>Submit</button>
<input type="text" />
```

### Анатомия HTML‑элемента
Каждый элемент обычно включает:
- **открывающий тег** — `<tag>`;
- **атрибуты** — `id`, `class`, `name`, `type` и другие;
- **содержимое** (если есть);
- **закрывающий тег** — `</tag>`.

Например:
```html
<button class="btn primary">Save</button>
```
- `<button>` — тег;
- `class="btn primary"` — атрибуты;
- `Save` — текстовое содержимое;
- `</button>` — закрывающий тег.

Понимание структуры HTML — основа для выбора корректного локатора.

## DOM
DOM (Document Object Model) — это дерево элементов веб‑страницы. Браузер преобразует HTML в структуру, напоминающую иерархию:

```text
html
└── body
    ├── header
    ├── main
    │   └── div
    │       ├── p
    │       └── button
    └── footer
```

Каждый узел — это элемент интерфейса, а вложенность показывает его положение.

### Почему DOM важен в автоматизации
- Поиск элементов выполняется **внутри DOM‑дерева**.
- Локатор описывает путь к нужному элементу.
- Автотесты взаимодействуют именно с DOM, а не с визуальным отображением страницы.

Если тест не может найти элемент, возможны причины:
- элемент отсутствует в DOM;
- элемент присутствует, но скрыт (`display: none`, `visibility: hidden`);
- структура или атрибуты изменились после загрузки страницы.

Если элемент визуально переместили, но его атрибуты остались прежними, тест всё равно сможет его найти. Однако при изменении `id`, `class` или структуры DOM локатор перестаёт работать.

## Локаторы
Локатор — это выражение, по которому инструмент автоматизации находит элемент на странице.
В Selenium, Selene, Playwright локатор включает стратегию поиска (ID, CSS, XPath и др.) и селектор.

### Зачем нужны локаторы
- **Основа теста:** любое действие начинается с поиска элемента.
- **Стабильность:** корректные локаторы уменьшают флейки.
- **Скорость:** чем проще локатор, тем быстрее поиск.
- **Поддерживаемость:** понятные локаторы проще читать и менять.
- **Диагностика:** информативный локатор упрощает поиск причины падения теста.

### Влияние на проект
- уменьшение нестабильности автотестов;
- сокращение стоимости изменений (1 исправление вместо десятков);
- ускорение ревью;
- корректные проверки рядом с понятными локаторами.
