# Часть 2. CSS‑селекторы
CSS-селекторы позволяют находить элементы на странице по тегам, классам, идентификаторам и значениям атрибутов.
Это один из самых удобных и быстрых способов локализации элементов в UI-автоматизации.

Поиск по CSS-селекторам основан на структуре HTML-документа и характеристиках элементов.
Браузер выбирает те узлы DOM, которые соответствуют указанному правилу селектора, — так же, как он применяет к ним CSS-стили. Это делает механизм поиска предсказуемым, быстрым и нативно поддерживаемым всеми современными инструментами тестирования.

## Базовый синтаксис CSS‑локаторов

### Основные типы селекторов
| Селектор | Описание | Пример | Найдёт |
|---------|----------|--------|--------|
| `tag` | По тегу | `input` | Все элементы `<input>` |
| `.class` | По классу | `.btn` | Все элементы с классом `btn` |
| `#id` | По идентификатору | `#submit` | Элемент с `id="submit"` |
| `[attr=value]` | По атрибуту | `[type="text"]` | Элементы с `type="text"` |

---

## Комбинирование селекторов
Комбинированные селекторы позволяют уточнять элемент по нескольким признакам.

| Синтаксис | Пример | Описание |
|-----------|--------|----------|
| `tag.class` | `button.btn` | Тег и класс |
| `tag#id` | `input#email` | Тег и id |
| `tag[attr=value]` | `input[name='username']` | Тег с атрибутом |
| `tag.class[attr=value]` | `button.btn[type='submit']` | Комбинация признаков |

---

## Иерархия элементов (родители и потомки)
CSS позволяет искать элементы с учётом вложенности.

| Оператор | Пример | Описание |
|----------|--------|----------|
| `A B` | `div input` | Потомок любого уровня |
| `A > B` | `form > input` | Только прямой потомок |
| `A + B` | `label + input` | Элемент сразу после `A` |
| `A ~ B` | `label ~ input` | Все элементы после `A` на том же уровне |

---

## Атрибуты и их вариации
| Селектор | Описание | Пример |
|----------|----------|---------|
| `[attr^=value]` | Значение начинается с… | `[id^='user']` |
| `[attr$=value]` | Значение заканчивается на… | `[class$='active']` |
| `[attr*=value]` | Значение содержит… | `[href*='login']` |

---

## Псевдоклассы
Псевдоклассы позволяют выбирать элементы по их положению или состоянию.

| Селектор | Описание | Пример |
|----------|----------|---------|
| `:first-child` | Первый ребёнок | `ul li:first-child` |
| `:last-child` | Последний ребёнок | `ul li:last-child` |
| `:nth-child(n)` | N‑й ребёнок | `table tr:nth-child(2)` |
| `:nth-of-type(n)` | N‑й элемент типа | `p:nth-of-type(3)` |
| `:not(selector)` | Исключение | `input:not([type='hidden'])` |
| `:checked` | Активный checkbox | `input:checked` |
| `:disabled` / `:enabled` | Заблокированные/активные | `button:disabled` |

---

## Примеры комбинирование разных селекторов
```css
button.btn.btn-primary[type='submit']:not(:disabled)
ul li:nth-child(2)[class*='item']
footer a[href^='http']
```

## Мини‑шпаргалка по CSS‑селекторам
| Цель | Селектор |
|------|----------|
| По тегу | `div` |
| По классу | `.class` |
| По id | `#id` |
| По атрибуту | `[attr='value']` |
| Частичное совпадение | `[attr*='part']` |
| Начало значения | `[attr^='start']` |
| Конец значения | `[attr$='end']` |
| Потомок | `A B` |
| Прямой потомок | `A > B` |
| Сосед | `A + B` |
| Исключение | `:not()` |
| По порядку | `:nth-child(n)` |
| По состоянию | `:checked`, `:disabled`, `:enabled` |
